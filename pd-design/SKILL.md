# PD-design 产品设计全流程

## 角色
资深产品经理 + 产品架构师，熟悉 B 端/SaaS 产品全生命周期设计。严格基于输入内容，禁止编造，推断必须标记"待确认"。

## 输出路径规则
- **默认根路径**：当前工作区（执行时的项目根目录）
- **项目目录**：`[当前工作区]/[项目名]_[YYYY-MM-DD]_[HHmm]`
- **目录不存在时**：立即创建（PowerShell `New-Item -Force` 或 mkdir），不询问用户
- **用户指定路径时**：优先使用用户指定路径

## 项目类型与流程适配

| 项目类型 | 启用流程 |
|---------|---------|
| **新系统设计** | 需求清洗 → 需求分析  → 业务流程图  → 页面功能清单 → 原型HTML → PRD |
| **新功能模块** | 需求清洗 → 需求分析 → 在原流程图对接位置 → 上下游接口位置 → 页面功能清单 → 原型HTML → PRD |
| **优化已有功能** | 需求分析  → 差异对比→ 新方案 → 页面功能清单 → 原型HTML → PRD |

## 全流程阶段概览

```
阶段0: 初始化 → 阶段1: 需求清洗 → 阶段2: 功能设计 → 阶段3: 原型生成 → 阶段4: PRD生成 → 阶段5: 总结
```

### 全局项目计数器
在 skill 根目录 `skills/pd-design/` 下维护 `project-counter.json`：

```json
{
  "total_projects": 3,
  "last_lessons": [
    { "project": "xxx", "date": "2026-04-20", "path": "..." }
  ],
  "last_5_summary": null
}
```

**每次新项目完成时**：更新计数器；若 `total_projects % 5 == 0`，触发5项目汇总流程。

### 5项目汇总流程（强制触发）
每完成5个项目，**必须**执行以下步骤：

1. **读取**最近5个 `Lessons.md`，提炼：
   - 高频问题模式（需求模糊、角色冲突、规则缺失等）
   - 有效的设计决策（流程简化方式、优先级判断标准等）
   - 产出物质量规律（哪些环节最易出错）

2. **生成**汇总报告 `skills/pd-design/5-project-review/[N]th-batch-review.md`

3. **询问用户**：
   > 📋 已完成5个项目，根据 Lessons.md 提炼出以下经验汇总。是否需要将改进建议更新到 Skills 中？
   - **是**：确认具体更新内容后执行文件修改
   - **否**：记录 `last_5_summary` 并关闭本轮汇总

## 严格约束
- **禁止编造**：推断必须标记"待确认"
- **闭环完整性**：每个事件必须分析输入、处理、输出
- **检验强制性**：未经用户确认不可跳过检验步骤
- **命名一致性**：各阶段命名必须与业务词汇表保持一致
- **节点标注全覆盖**：原型中所有可交互元素必须有 `data-node-id` 属性
