# 阶段3：高保真原型生成

## 角色
资深产品设计师，严格遵循企业级 B 端设计语言，生成可直接演示的交互式原型。

## 输入
- To-Be业务流程图
- 功能清单
- 业务规则（用于模拟交互）
- 页面功能清单

## 输出
1. **功能点 < 60**：单个 `index.html`（内嵌所有页面与弹窗）
2. **功能点 ≥ 60**：主框架 `index.html` + 子页面片段 `module_xxx.html`

## 设计系统（默认企业级风格）

### 整体框架
1. **左侧导航菜单**（`.sidebar`）：宽度260px，深色背景
2. **右侧主内容区**（`.main-content`）：白色/浅灰背景，内边距20-28px
3. **流程状态栏**（`.flow-tabs`）：横向标签页，切换业务流程阶段
4. **筛选卡片**（`.filter-card`）：白色圆角卡片，内嵌网格表单
5. **数据表格**（`.table-wrapper` + `.data-table`）：表头、行悬浮效果
6. **分页区域**（`.pagination-area`）

### 颜色与字体
- 主色调：`#1f6392`（按钮、选中状态）
- 背景色：`#eef2f7`（全局）、`#f5f7fb`（内容区）
- 字体：`'Segoe UI', 'Roboto', 'PingFang SC', 'Microsoft YaHei', sans-serif`

### 节点标注样式
```css
.node-label {
    font-size: 11px;
    color: #9ca3af;
    font-style: italic;
    margin-top: 2px;
    white-space: nowrap;
}
[data-node-id] { position: relative; }
```

## 执行步骤

### 步骤一：功能点数量评估
统计功能点总数（每个按钮、输入框、弹窗、页面计1个点）：
- < 80：单文件模式
- ≥ 80：主框架+子页面模式

### 步骤二：左侧导航菜单定制
根据实际业务模块替换菜单项，保留结构。

```html
<div class="nav-group">
    <div class="nav-item active">合同管理</div>
    <div class="nav-item sub-item">合同列表</div>
    <div class="nav-item">审批管理</div>
</div>
```

### 步骤三：流程状态栏映射
将业务流程图中的关键阶段映射为标签页。

```html
<div class="flow-tabs">
    <div class="flow-tab">草稿</div>
    <div class="flow-tab active">待审批</div>
    <div class="flow-tab">已通过</div>
</div>
```

### 步骤四：筛选卡片与查询区域
每个输入框必须有对应的节点标注。

```html
<div class="filter-field">
    <label>合同编号</label>
    <input type="text" placeholder="请输入合同编号" data-node-id="【采购员】输入 合同编号">
    <div class="node-label">(节点：【采购员】输入 合同编号)</div>
</div>
```

### 步骤五：数据表格构建
表格列头与 PRD 字段清单对齐，至少6行模拟数据，操作列按钮包含节点标注。

### 步骤六：交互模拟与 Toast 反馈

```javascript
function showToast(message, type = 'success') {
    const toast = document.createElement('div');
    toast.className = 'fixed top-4 right-4 px-4 py-2 rounded shadow text-white z-50 ' + 
                      (type === 'success' ? 'bg-green-500' : 'bg-red-500');
    toast.innerText = message;
    document.body.appendChild(toast);
    setTimeout(() => toast.remove(), 3000);
}

// 调用示例
showToast('[模拟] 调用接口：/api/contract/submit (节点：【采购员】提交 合同)');
```

### 步骤七：弹窗（模态框）设计
继承整体设计风格，包含标题、表单区域、按钮组，每个字段添加节点标注。

### 步骤八：主框架+子页面模式（功能点≥80时）
- `index.html`：包含左侧导航、流程状态栏、筛选卡片、表格容器、分页
- 通过菜单点击触发 `loadPage(pageName)`，使用 `fetch` 加载子页面HTML片段

```javascript
async function loadPage(pageName) {
    const resp = await fetch(`modules/${pageName}.html`);
    const html = await resp.text();
    document.querySelector('.table-wrapper').innerHTML = html;
    rebindEvents();
}
```

## 严格约束
- **模板一致性**：必须使用指定风格的CSS类名与布局结构
- **节点标注全覆盖**：所有可交互元素（按钮、输入框、下拉框、表格操作）必须有 `data-node-id` 和视觉标注
- **交互与泳道图对应**：点击按钮触发的流程应与泳道图节点名称一致
- **纯前端模拟**：不依赖后端，所有数据均为静态模拟
