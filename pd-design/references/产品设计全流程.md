# 产品设计全流程总览

## 适用项目类型与流程适配

| 项目类型 | 启用流程 |
|---------|---------|
| **新系统设计** | 需求清洗 → 需求分析 →  业务流程图 → 页面功能清单 → 原型HTML → PRD |
| **新功能模块** | 需求清洗 → 需求分析 →  在原流程图对接位置 → 上下游接口位置 → 页面功能清单 → 原型HTML → PRD |
| **优化已有功能** | 需求分析 →  差异对比→ 新方案 → 页面功能清单 → 原型HTML → PRD |

## 进度管理机制

### 状态文件
在工作区目录下自动创建 `.prd-progress.json`，记录当前执行状态，每次阶段完成后立即更新。

### 启动时检查
- 若工作区存在 `.prd-progress.json`，读取并显示已完成阶段，询问用户：
  > 检测到未完成的PRD项目，当前进度：已完成阶段0、1。是否从阶段2继续？（y/n）
- 若用户选择继续，则从 `current_stage` 开始执行；若选择重新开始，则归档旧进度文件并新建。

### 超时恢复
若会话超时中断，用户再次输入"继续"或"进度"时，Agent 自动读取进度文件并从中断处继续。

## 阶段映射

| 阶段 | 名称 | 核心产出 |
|------|------|---------|
| 0 | 初始化 | 创建进度文件，确定项目类型和工作区 |
| 1 | 需求清洗与冲突仲裁 | 需求分析报告、口语化整理稿、业务事件清单、As-Is流程图 |
| 2 | 业务建模与功能设计 | To-Be流程图、用例图、状态机、时序图、RBAC、功能清单 |
| 3 | 高保真原型生成 | index.html（可交互HTML原型） |
| 4 | PRD生成与文档校验 | PRD.md + PRD.docx，校验报告 |
| 5 | 总结 | 经验沉淀 Lessons.md |

## 阶段5：总结与 Lessons.md 生成

详见 [references/阶段5总结.md](references/阶段5总结.md)

**强制产出物：**
- `Lessons.md`（经验教训记录）
- 更新 `skills/pd-design/project-counter.json` 计数器
- 若 `total_projects % 5 == 0`，触发5项目汇总流程

## 交付完成
- 进度文件标记为 `completed: true`，归档到输出目录的 `meta/` 下
- 生成包含经验教训的 Lessons.md 文件
